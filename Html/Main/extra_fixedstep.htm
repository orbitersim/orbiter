Fixed time steps

This dialog allows to put Orbiter into fixed time interval mode. In this mode, the simulation advances by a predefined interval at each time step, independently of how long the calculation of the frame took in real time.

Fixed time intervals are good for debugging problems, because the simulation state after a given number of steps does not depend on the speed of the computer Orbiter is running on, so it is easier to compare the results on different machines.

Fixed steps can also be used to generate trajectories with consistent accuracy (for example when recording a mission, or for testing/optimising a flight path).

The disadvantage of fixed time steps is the fact that the simulation time is no longer equal to real time (not even to a fixed multiple of real time). The simulation will progress fast when the frame rate is high, but slowly when the frame rate is low - so Orbiter no longer works in "real time" mode.

You can still use the time acceleration keys (T and R) in fixed interval mode. Orbiter simply multiplies the step length by the warp factor when calculating the frame time interval.

To switch to fixed interval mode, select Enable fixed time step length in the dialog box, then enter the time interval per frame (in seconds).

Note: If you run Orbiter in fullscreen mode, and do not disable vertical sync., then by selecting a fixed time step that is the inverse of the monitor's refresh rate, Orbiter will run in real time as long as the time required for calculating and rendering a frame does not exceed the refresh interval.
